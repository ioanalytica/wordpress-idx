# wordpress-idx — Iteration Plan

## Iteration 1 — Project scaffold & DB connection
- Initialize Node.js project (package.json, dependencies)
- Environment config (`.env.example`, config module reading `DB_HOST`, `DB_USER`, `DB_PASS`, `DB_NAME`, `DB_PREFIX`, `PORT`)
- `DATA_DIR` fixed to `/data`
- Connect to WordPress MySQL database, verify connection
- Basic Express server with a `/healthz` endpoint

## Iteration 2 — WordPress data extraction
- Query WordPress DB for posts/pages with all required metadata (author, title, date, categories, tags, content, slug)
- WordPress stores categories/tags in a taxonomy system across multiple tables — proper JOINs required
- Write the result as `wp-index-source.json` to `/data`

## Iteration 3 — FlexSearch index creation
- Read `wp-index-source.json`, build a FlexSearch index
- Export/serialize the index as `wp-index-flex.json`
- Delete `wp-index-source.json` after successful index creation

## Iteration 4 — Search API
- On startup: if `wp-index-flex.json` exists, load it into FlexSearch
- `GET /api/search` endpoint with query parameters for:
  - Metadata filtering (type, author, category, tag, date range)
  - Phrase/word search in content
- Response: matching posts with metadata + contextual snippet (up to 500 chars with ±200 chars around match)

## Iteration 5 — Operational polish
- Dockerfile
- `POST /api/reindex` endpoint to trigger a fresh extraction + index rebuild
- Error handling, logging, graceful shutdown
- Review `.gitlab-ci.yml` integration
